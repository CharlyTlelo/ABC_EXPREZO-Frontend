Estándares Frontend – Documentación Actual (E-Commerce FrontEnd)

Framework: Angular 11.x UI: PrimeNG 11, PrimeFlex, PrimeIcons; Google Maps; DataTables; SweetAlert2 Ámbito: Documentación de lo existente (sin recomendaciones ni cambios).

1) Estructura del proyecto

  • Módulos feature con su *-routing.module.ts.
  • Componentes organizados por dominio funcional (ej. articles, clients, orders).
  • Servicios por feature para llamadas HTTP.
  • Re-exports de UI desde un SharedModule (PrimeNG/Directivas comunes).

2) Enrutamiento

  • Rutas en kebab-case; lazy loading donde aplique.
  • Rutas declaradas en módulos feature y referenciadas desde app-routing.
  • Inventario de rutas disponible como CSV/JSON generado (si se adjunta en assets/data).

3) Scripts de NPM (package.json)

  • start: ng serve
  • build: ng build (con perfiles dev/prod en --configuration)
  • test: ng test
  • lint: ng lint
  • e2e: ng e2e
  • postinstall: ngcc

4) Dependencias y UI

Angular & ecosistema (ver package.json)

  • @angular/* 11.x, @angular/cdk 11.x

UI / Componentes

  • PrimeNG: primeng ^11.x (button, table, dialog, dropdown, toast, tooltip, etc.)
  • Google Maps: @agm/core, @angular/google-maps
  • DataTables: angular-datatables, datatables.net*, jquery
  • SweetAlert2: @sweetalert2/ngx-sweetalert2, sweetalert2
  • Angular Material: si existe @angular/material (uso mixto con PrimeNG).

Estilos globales (angular.json)

  • src/styles.css
  • Temas PrimeNG/PrimeFlex/PrimeIcons
  • Hojas de DataTables

5) HTTP & RxJS

  • HttpClient con baseURL desde environment.*.
  • Encapsular llamadas en servicios; manejar errores de forma centralizada.
  • Desuscripción con async pipe o patrón takeUntil en OnDestroy.

6) Environments (sin valores)

  • Claves típicas: url_api, S3 (bucket/region/identity), Google Maps, tokenCopomex, etc.
  • Separación environment.ts / environment.prod.ts.

7) Fichas técnicas (componentes)

  • Responsabilidad, Inputs/Outputs, servicios usados, navegación/UX, errores y mensajes, dependencias UI.
  • Accesibilidad: aria-*, roles; textos legibles.

8) Resumen

  • Angular 11 + PrimeNG, DataTables, Google Maps, SweetAlert2.
  • Rutas por dominio funcional; inventario exportable (CSV/JSON).
  • Environments documentados por claves (sin exponer valores).
